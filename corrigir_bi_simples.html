❌ Erro Atual: "Erro ao montar o BI: Erro no navegador (sobrecarga)"

📚 O que é o erro?

O navegador está recebendo MUITOS DADOS da API e não consegue processar tudo. É como tentar carregar 1000 coisas no carro pequeno. O carro (navegador) não aguenta e trava.

1

O Problema:

A página BI está tentando carregar dados de 500+ veículos de uma vez ❌

2

A Solução:

Limitar para 3 meses de dados, 20 veículos no consumo e 10 nos rankings ✅

🚀 Como Corrigir? (3 opções)

💡 Dica: Comece pela Opção 1 (Automática) se você for admin. Se não for, use a Opção 1.5. É a mais fácil!
A

Esta é a forma MAIS FÁCIL. Clique no botão abaixo e deixa pronto:

✅ CLIQUE AQUI PARA CORRIGIR

  • 1️⃣ Você vai para uma página

OPÇÃO 2: Correção Manual (MySQL)

Se a opção 1 não funcionar, use esta. Abra o phpMyAdmin:

Passo 1: Acesse:

http://localhost/phpmyadmin

Passo 2: Cole este código:

CREATE INDEX IF NOT EXISTS idx_desp_viagem_rota_empresa ON despesas_viagem(rota_id, empresa_id); DELETE FROM bi_cache WHERE 1=1;

Passo 3: Clique em "Executar" (ou Enter)

Passo 4: Recarregue o BI com Ctrl+F5

C

OPÇÃO 3: Leitura do Código (Para Desenvolvedores)

  • 📄 api/performance_indicators.php - Limita dados retornados
  • 📄 pages/bi.php - Limita renderização no JS

✅ Após Corrigir, O Que Fazer?

1️⃣ Recarregue a Página

Pressione: Ctrl + F5 (ou Cmd + Shift + R no Mac)

Isso "limpa" a cache do navegador

2️⃣ Acesse o BI Novamente

Clique em BI Frota no menu lateral

3️⃣ Verifique Se Funcionou

Você deve ver:

  • ✅ Gráficos carregando
  • ✅ Tabelas aparecendo
  • ✅ Nenhuma mensagem de erro
  • ✅ Animações suaves no carregamento
  • ✅ Atalho Ctrl+R para atualizar dados

❓ E Se Ainda Houver Erro?

Se a página de BI CONTINUAR dando erro, faça isso:
1

Use o Diagnóstico

Clique no link abaixo para verificar quanto dados você tem:

🔍 ABRIR DIAGNÓSTICO
2

Use o Teste da API

Para ver o tamanho exato da resposta da API:

🧪 ABRIR TESTE DA API
3

Procure por "⚠️ MUITOS DADOS"

Use o diagnóstico para verificar o volume de dados:

🔍 ABRIR DIAGNÓSTICO

Se encontrar "⚠️ MUITOS DADOS", significa que sua empresa tem MUITO volume de dados (tabelas com 5000+ registros).

4

Verifique o Console do Navegador

Pressione F12, vá na aba Console e recarregue a página. Procure por erros em vermelho.

Se aparecer "Maximum call stack size exceeded" ou similar, ainda há sobrecarga de dados.

5

Próximas Ações (Se Necessário)

Se o volume for muito alto, as soluções são:

  • 📈 Arquivar dados antigos
  • 📊 Separar BI em múltiplas páginas
  • ⚙️ Implementar filtros mais restritivos
  • 📧 Contactar suporte técnico
